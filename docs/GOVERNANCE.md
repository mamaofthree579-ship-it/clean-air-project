Governance Model

This project uses a **lightweight, contributor-driven governance structure**.

## Roles

### Maintainers
- Approve PRs
- Review security reports
- Oversee releases
- Coordinate community decisions

### Contributors
- Submit features, fixes, documentation, or hardware designs
- Participate in testing and reviews
- Propose new initiatives

### Advisors (optional)
- Provide domain expertise in sensors, air quality science, public health, or community deployments

## Decisions
We use:
- **Consensus** for small changes  
- **Maintainer vote** for large decisions  
- **Public proposals** via Discussions for roadmap items
